# E2E 점검 체크리스트 (전체 플로우 · 최신 반영분 포함)

> 실제 사용 순서대로 **본사 → 원장 → 디자이너 → 자동화/보안**을 처음부터 끝까지 눌러보며 확인하는 체크리스트예요.
> 지점 하나를 새로 열 때, 또는 배포 후 "다 도는지" 확인할 때 이 문서를 위→아래로 따라가면 돼요.
> 🆕 = 6월 말~7월 초에 **새로 반영된 것**.
>
> _기준일: 2026년 7월 1일 (6/30 스프린트 반영)_

---

## 🔎 이번에 새로 반영된 것 (먼저 눈여겨볼 항목)

- 🆕 **릴스 AI** — 영상 분석 + 컷별 대본 생성 + 링크 추적 (`/reels`, `api/reels/*`)
- 🆕 **내 콘텐츠 프로필** — 릴스·글 품질 올리는 프로필 저장 (`api/reels/profile`)
- 🆕 **HandSOS 성과 자동 수집** — 매일 06:00 KST 크론, PC 안 켜도 됨 (`crawl-handsos.yml`)
- 🆕 **성과 대시보드 빠른 새로고침** — "어제만" 빠르게 수집(디자이너 스킵) (`performance/refresh`)
- 🆕 **회원 바로 추가** — 초대 없이 계정 즉시 발급(임시비번 `visual1234`) (`DirectMemberForm`)
- 🆕 **디자이너 개인 네이버 블로그 등록** — 사람별 발행 대상 (`MyNaverBlogField`, `api/me/blog`)
- 🆕 **GPS 출근 · 캡처 억제/워터마크** — 근태 + 유출 추적

## 0. 사전 확인 (배포·접속)

- [ ] 앱 주소 접속 시 **로그인 화면**이 뜬다
- [ ] `/api/health` 가 정상 응답한다 (서버 살아있음)
- [ ] 로그인 안 한 상태로 `/write` 등 내부 주소 직접 입력 → **로그인으로 튕긴다**

## 1. 본사 세팅 E2E _(본사 계정)_

### 1-1. 지점 만들기 — 「지점 관리」
- [ ] 「지점 추가」로 **이름**(필수) 입력 → 저장되고 목록에 뜬다
- [ ] **지역 · 지식베이스 slug** 입력이 저장된다
- [ ] **아임웹 글쓰기 링크(지점 공용)** 입력이 저장된다
- [ ] **「📍 현재 위치로 설정」** → 위도/경도 자동 입력된다 (매장 현장에서)
- [ ] **허용 반경**(기본 200m) 값이 저장된다
- [ ] 저장 후 **지점 수정**(연필) → 값이 그대로 불러와지고 재저장된다
- [ ] 지점 목록에 **멤버 수 · 글 수**가 보인다

### 1-2. 원장 계정 만들기 — 「지점·사람」 → 「바로 추가」 🆕
- [ ] **이름 · 휴대폰 · 임시비번(`visual1234`) · 역할=원장 · 지점** 선택 후 「바로 추가」
- [ ] "계정이 만들어졌어요" + **아이디(휴대폰)·비번**이 표시된다
- [ ] **「아이디·비번 복사」** 가 동작한다
- [ ] 같은 휴대폰 재등록 시 **"이미 가입된 휴대폰 번호예요"** 로 막힌다
- [ ] (초대 방식) 「새로 초대하기」로 링크 생성 → **「초대 링크 보내기」** 공유 동작
- [ ] **수락 대기 초대**(Pending) 목록에 뜨고, 취소가 된다

### 1-3. 전달
- [ ] 만든 **아이디·비번**을 원장에게 전달(안내문 문구 사용)

## 2. 원장 E2E _(원장 계정으로 로그인)_

- [ ] 발급받은 **휴대폰 아이디 + `visual1234`** 로 로그인된다
- [ ] 좌측/하단 메뉴가 **원장용(7개)** 으로 보인다 (전체현황·지점관리·본사메뉴는 안 보임)

### 2-1. 디자이너/인턴 등록 — 「우리 지점 사람」
- [ ] **「바로 추가」** 로 디자이너 계정 발급 → 아이디·비번 표시/복사 🆕
- [ ] **「새로 초대하기」** 로 초대 링크 생성 → 공유
- [ ] 역할 선택이 **디자이너/인턴만** 나온다 (원장·본사 옵션 없음)
- [ ] 다른 지점 사람은 **목록에 안 보인다**
- [ ] 멤버의 **역할 변경(디↔인턴)** / **비활성화(내보내기)** 가 된다
- [ ] 각 멤버의 **이번 달 글 수**가 표시된다

### 2-2. 성과 대시보드 — 「성과 대시보드」
- [ ] 우리 지점 매출 **전체/신규/재방문** 카드 + 전월 대비 ↑↓ 가 보인다
- [ ] **객단가 · 복구매직 비율 · 시술 구성**(건수)이 보인다
- [ ] **"고객이 우리한테 오기까지"** 퍼널(노출→방문)과 **AI 진단 카드**가 뜬다
- [ ] **「새로고침」** → "어제" 데이터 수집 완료 메시지가 뜬다 🆕
- [ ] **월간/주간** 전환이 된다
- [ ] HandSOS 미연동 지점은 **"연동 전"** 으로 표시된다

### 2-3. 범위 확인
- [ ] 「출근」·「내 글·조회수」에서 **"우리 지점" 범위**로 볼 수 있고 **엑셀 내보내기**가 된다

---

## 3. 디자이너 E2E _(디자이너 계정으로 로그인)_

- [ ] 휴대폰 아이디 + 비번으로 로그인 → 메뉴가 **디자이너용(5개)** 로 보인다
- [ ] 홈 화면에 **"OO 디자이너님 👋"** + 지난 글 목록이 보인다

### 3-1. 첫 세팅 — 네이버 블로그 등록 🆕
- [ ] 「글쓰기」에서 **내 네이버 아이디**만(`blog.naver.com/` 뒤) 등록 → 저장된다
- [ ] 이후 **「네이버 블로그 열기」** 가 내 글쓰기 창을 연다

### 3-2. 글쓰기 전체 플로우 — 「글쓰기」
- [ ] **시술 칩**(결마지·펌·염색·클리닉·컷) 다중 선택된다
- [ ] **「말로 쉽게 설명하기」 녹음** → 텍스트로 변환된다 (`api/transcribe`)
- [ ] **사진 1~2장** 업로드된다
- [ ] **「추천 받기」** → 주제 2~3개 생성 (`api/recommend-topics`)
- [ ] **「이 주제로 글쓰기」** → 오른쪽에 초안 생성 (`api/generate`)
- [ ] **「고쳐쓰기」** 로 다시 생성된다
- [ ] **「네이버 블로그 열기」/「아임웹 열기」** → 내용 자동복사 + 사이트 열림
- [ ] 초안이 **저장**되어 「내 글·조회수」/홈 목록에 뜬다 (`api/posts`)

### 3-3. 릴스 전체 플로우 — 「릴스」 🆕
- [ ] **「내 콘텐츠 프로필」** 채우고 저장 → 유지된다 (`api/reels/profile`)
- [ ] **참고 영상 업로드 → 「영상 올려서 분석」** → 훅/이유/자막 예시 (`api/reels/analyze`)
- [ ] 20MB 초과 영상은 **막히거나 안내**된다
- [ ] 시술 + 메모 + **앵글(담백/욕망)** 선택 후 **「릴스 구성 만들기」** → 컷별 대본 (`api/reels/generate`)
- [ ] **「대본·자막 복사」 / 「인스타 열어서 올리기」** 동작
- [ ] **올린 릴스 주소 붙여넣고 「링크 등록하고 추적 시작」** → 등록됨 (`api/reels/[id]`)

### 3-4. 리뷰 답글 — 「리뷰 답글」
- [ ] 고객 리뷰 붙여넣기 + (시술칩) → **「답글 써드릴게요」** → 답글 2~3개 (`api/review-reply`)
- [ ] **「복사하고 스마트플레이스 열기」** 동작 (저장 안 됨, 일회성)
- [ ] 추천 키워드가 답글에 반영된다

### 3-5. 내 글·조회수 — 「내 글·조회수」
- [ ] 상단 **이번 달 글 수 · 총 조회수 · 평균** 카드가 맞다
- [ ] 글의 **「조회수 입력」** → 주소 + 조회수 입력 후 **「기록하기」** 저장 (`ViewsForm`, `api/posts` PATCH)
- [ ] **「3일 뒤에 다시 물어볼게요」** 켜면 3일 후 리마인드로 다시 뜬다

### 3-6. 출근 — 「출근」 🆕
- [ ] **매장 반경 밖**에서는 출근이 **막힌다**(거리 안내)
- [ ] 출근 시 **그루밍 4종**(명찰·무전기·메이크업·헤어) 전부 체크해야 진행된다
- [ ] **출근 사진** 첨부가 필수다
- [ ] **「출근하기」** → 기록되고, 이후 **외출/복귀/퇴근** 버튼으로 상태가 바뀐다
- [ ] 「출근」 목록에서 **본인 기록만** 보인다

## 4. 본사 전용 화면 E2E _(본사 계정)_

### 4-1. 전체 현황 — 「전체 현황」
- [ ] 로그인 첫 화면이 **전체 현황**이다
- [ ] **지점·사람·글·총 조회수** KPI가 집계된다
- [ ] 지점별 표에서 **원장 없는 지점 ⚠️** 경고가 뜬다
- [ ] 지점명 클릭 → 그 지점 **사람 관리**로 이동

### 4-2. 키워드 조사 — 「키워드 조사」
- [ ] **월별 엑셀 업로드** → 지점별 탭 생성 (`api/keyword-research`)
- [ ] 키워드 **추천 토글** ON/OFF → 저장 (`api/keyword-research/[id]`)
- [ ] 반영 후 **글쓰기/리뷰 답글 추천**에 실제로 반영된다

### 4-3. 아카데미 마케팅 — 「아카데미」
- [ ] 아임웹 마케팅 **엑셀 업로드** → 방문→회원→구매 퍼널 표시 (`api/academy`)
- [ ] **유입경로** 그래프가 뜬다
- [ ] **월간/주간** 전환이 된다

### 4-4. 본사 권한
- [ ] 「지점·사람」에서 **원장/본사 계정**도 발급/역할변경 가능
- [ ] 멤버 **완전 삭제**(본사만)가 동작한다

## 6. 보안 · 유출 추적 확인 _(전 계정 자동)_

- [ ] 모든 화면에 **워터마크**(이름·지점·번호 뒤4자리)가 은은히 깔린다 (`Watermark`)
- [ ] **스크린샷/녹화 억제**가 동작한다 (`CaptureGuard`)
- [ ] 민감 화면(예: `/track/[id]`, 「지점·사람」) 접근 시 **접근 로그**가 남는다
- [ ] 사진 열람은 **한시적 서명 URL**로만 열린다(영구주소 아님)

## 7. 권한 경계 테스트 (중요 — 역할별 차단 확인)

- [ ] **디자이너**로 `/performance` 직접 접근 → **막히거나 「내 글·조회수」로 리다이렉트**
- [ ] **디자이너**로 `/overview` `/branches` `/members` 접근 → **차단**
- [ ] **원장**으로 `/overview` `/branches` `/keyword-research` `/academy` 접근 → **차단**
- [ ] **원장**이 다른 지점 멤버/원장/본사를 **관리 불가**
- [ ] **비활성화된 계정**은 로그인해도 내부 접근이 **차단**된다

## ✅ 최종 스모크 요약 (딱 5개만 빠르게)

1. [ ] 본사 → 지점 생성 + 원장 「바로 추가」 → 아이디·비번 발급
2. [ ] 원장 로그인 → 디자이너 「바로 추가」 → 성과 대시보드 「새로고침」
3. [ ] 디자이너 로그인 → 네이버 블로그 등록 → **글쓰기 초안 생성 → 열기**
4. [ ] 디자이너 → **릴스 구성 생성** + **출근(그루밍+사진)**
5. [ ] 크론(HandSOS/글생성) 수동 실행 → 대시보드·목록 반영
