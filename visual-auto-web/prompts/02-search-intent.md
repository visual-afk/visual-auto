# 프롬프트 2: 검색 의도 분석 + 콘텐츠 개요

너는 네이버 블로그 SEO 전략가야.
아래 메인 키워드를 검색하는 사람이 뭘 알고 싶어하는지 분석하고, 그에 맞는 블로그 글 뼈대(소제목 구조)를 만들어줘.

메인 키워드: {{keyword}}

---

## [STEP 1] 이 키워드를 검색하는 사람은 누구야?

아래 3가지를 구체적으로 분석해:

### 1-1. 검색자의 상황
> 이 사람이 지금 어떤 상황이길래 이걸 검색했을까?

예시:
- "펌을 하고 싶은데 어떤 펌이 나한테 맞을지 모르겠다"
- "펌을 했는데 생각보다 빨리 풀려서 당황하고 있다"

### 1-2. 해결하고 싶은 문제
> 이 사람이 진짜 알고 싶은 건 뭐야? (겉으로 드러난 질문 + 숨겨진 궁금증)

| 구분 | 설명 |
|------|------|
| 겉 질문 | 검색어에 그대로 드러나는 궁금증 |
| 속 질문 | 검색어엔 없지만 같이 궁금할 것들 |

예시:
- 겉: "볼륨펌 유지 기간이 얼마나 되지?"
- 속: "내 머리 상태로 했을 때도 그만큼 가나?", "유지하려면 어떻게 관리해야 하지?"

### 1-3. 검색 의도 분류

이 키워드의 검색 의도를 아래 중에서 골라:

| 의도 | 뜻 | 예시 키워드 |
|------|----|------------|
| 정보 탐색 | 궁금한 걸 알고 싶음 | "볼륨펌 유지 기간" |
| 비교 선택 | 뭘 고를지 결정하고 싶음 | "볼륨펌 vs 셋팅펌" |
| 문제 해결 | 불만/걱정을 해결하고 싶음 | "펌 빨리 풀리는 이유" |
| 방문 탐색 | 미용실을 찾고 있음 | "강남 볼륨펌 잘하는 곳" |

---

## [STEP 2] 경쟁 글 분석

이 키워드로 **웹검색**을 해서, 네이버 상위 노출 글들이 어떤 내용을 다루는지 확인해.

아래 표로 정리해:

| 순위 | 글 제목 | 다루는 주요 내용 | 빠져있는 내용 |
|------|---------|-----------------|--------------|
| 1 | (제목) | (어떤 내용) | (안 다루는 것) |
| 2 | (제목) | (어떤 내용) | (안 다루는 것) |
| 3 | (제목) | (어떤 내용) | (안 다루는 것) |

그리고 한 줄로 정리해:
> **차별화 포인트:** 상위 글들이 안 다루는데 우리가 다루면 좋은 내용은 "○○○"

---

## [STEP 3] 콘텐츠 개요 (소제목 구조) 작성

STEP 1의 검색 의도 + STEP 2의 차별화 포인트를 반영해서 블로그 글의 뼈대를 만들어.

### 작성 규칙

1. **H2(큰 소제목)은 3~6개**
2. **H3(작은 소제목)은 꼭 필요할 때만** (H2 하나에 H3은 2~3개 이하)
3. **소제목은 질문 형태가 좋음** (예: "볼륨펌 유지 기간은?")
4. **사람들이 실제로 검색하는 말투** 사용 (전문 용어 ❌)
5. **키워드를 소제목에 자연스럽게 포함**
6. **글의 흐름**: 궁금증 해결 → 비교/선택 도움 → 비주얼살롱 소개 순서

### 출력 형식

```
# H1: (글 제목)

## H2-1: (소제목) — [정보형/비교형/홍보형]
  ### H3-1-1: (소제목)
  ### H3-1-2: (소제목)

## H2-2: (소제목) — [정보형/비교형/홍보형]
  ### H3-2-1: (소제목)

## H2-3: (소제목) — [정보형/비교형/홍보형]

...

## 마무리: (예약 유도)
```

각 H2 옆에 [정보형/비교형/홍보형] 태그를 붙여줘.
(→ 3번 단계에서 이 태그를 보고 비율을 체크함)

---

## [STEP 4] 개요 요약 카드

3번 단계로 넘기기 좋게, 아래 요약도 같이 출력해:

```
[개요 요약]
- 메인 키워드: ○○○
- 검색 의도: (정보 탐색 / 비교 선택 / 문제 해결 / 방문 탐색)
- 글 유형: (정보형 / 스토리형 / 시즌형)
- H2 개수: ○개 (정보형 ○, 비교형 ○, 홍보형 ○)
- 타겟: (이 글을 읽을 사람)
- 차별화 포인트: (상위 글 대비 우리가 다를 점)
```

---

## 꼭 지켜야 할 것

1. **한국어**로 써
2. STEP 2에서 **반드시 웹검색**해서 실제 상위 글을 참고해
3. 레퍼런스(참고한 글)를 알려줘
4. 불필요한 서론 없이 바로 시작
5. 결과는 마크다운 파일 1개로 출력 (파일명: `02-search-intent.md`)
