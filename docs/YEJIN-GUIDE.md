그러면 Claude Code가 남은 세팅 단계를 하나씩 안내해줍니다.

## 일상 업무

### 이번주 일정 확인하기

> "이번주 블로그 일정 보여줘"

→ 이번주에 예정된 주제, 상태, 독스 링크가 보입니다.

### 새로운 주제 등록하기

> "4월 3주차에 '봄 웨이브펌 추천' 주제 넣어줘. 키워드는 웨이브펌, 봄펌, 디지털펌이고 스토리형이야. 예정일은 4월 21일"

→ 구글시트에 행이 추가되고, 캘린더에도 반영됩니다.

### 이번 달 기획서 한꺼번에 넣기

> "4월 기획서 넣어줘:
> - 2주차(4/14): 봄철 두피케어 가이드 / 키워드: 두피케어, 봄 헤어관리 / 정보형
> - 2주차(4/16): 웨이브펌 비포애프터 / 키워드: 웨이브펌, 디지털펌 / 스토리형
> - 3주차(4/21): 여름 앞두고 염색 트렌드 / 키워드: 여름염색, 하이라이트 / 시즌형"

→ 한 번에 여러 주제를 시트에 등록해줍니다.

### 블로그 글 생성하기

> "오늘 예정된 글 생성해줘"

또는 특정 주제를 지정:

> "'봄철 두피케어' 글 생성해줘"

→ AI가 글을 쓰고 구글독스에 올립니다. 캘린더에서 독스 링크를 확인하세요.

### 글 다시 생성하기

> "봄철 두피케어 글 마음에 안 들어. 다시 써줘"

→ 시트 상태를 planned로 되돌리고 다시 생성합니다.

### 시트 ↔ 캘린더 동기화

> "시트 내용 캘린더에 반영해줘"

→ 시트의 모든 주제가 캘린더 이벤트로 업데이트됩니다.

## 지식베이스(뇌) 채우기

AI가 좋은 글을 쓰려면 비주얼살롱에 대한 지식이 필요합니다.
아래처럼 Claude Code에게 말하면서 채우면 됩니다.

### 브랜드 보이스 설정

> "브랜드 보이스 파일 채워줘. 
> 우리는 전문적이면서도 친근한 톤이야. 
> 존댓말 쓰는데 ~해요 체. 
> 과장된 광고 문구는 절대 안 써. 
> 예를 들면 '무조건 최고' 이런 거 안 쓰고, '고객님의 모발 상태에 맞는 최적의 시술을 제안드려요' 이런 느낌"

→ Claude Code가 `knowledge/brand/brand-voice.md`를 작성해줍니다.

### 서비스 메뉴 등록

> "서비스 메뉴 파일 채워줘.
> 커트: 여성커트 3만원/40분, 남성커트 2만원/30분
> 펌: 디지털펌 15만원/2시간, 볼륨매직 12만원/2시간
> 염색: 전체염색 8만원~/1.5시간, 뿌리염색 5만원/1시간
> ..."

→ Claude Code가 `knowledge/brand/services-menu.md`를 정리해줍니다.

### 고객 사례 추가

> "고객 사례 하나 추가해줘.
> 30대 직장인 여성, 출산 후 탈모 고민이었어.
> 볼륨펌 + 두피케어 3회 진행.
> 결과: 볼륨감 살아나서 만족. 3개월째 정기 관리 중"

→ `knowledge/brand/case-studies.md`에 추가됩니다.

### 키워드 추가

> "키워드 목록에 '강남 남자펌 잘하는 곳', '남자 가르마펌 추천' 추가해줘"

→ `knowledge/seo/keyword-master.md`에 추가됩니다.

### AI가 먼저 써주는 파일

아래 파일들은 AI가 초안을 써줄 수 있습니다:

> "소비자 의사결정 여정 파일 초안 써줘"

> "CTA 패턴 파일 초안 써줘"

> "검색 의도 매핑 파일 초안 써줘"

→ AI가 일반적인 미용업 기준으로 초안을 쓰면, 예진매니저가 비주얼살롱에 맞게 수정합니다.

## 수정한 내용 반영하기 (Git)

knowledge/ 파일을 수정하면 반영해야 합니다. 이것도 Claude Code에게 말하면 됩니다:

> "수정한 내용 올려줘" 또는 "PR 올려줘"

→ Claude Code가 알아서:
1. `yejin/작업내용` 브랜치를 만들고
2. 변경사항을 커밋하고
3. GitHub에 PR(Pull Request)을 올립니다
4. 대표님한테 Google Chat 알림이 갑니다
5. 대표님이 확인하면 반영됩니다

**절대로 직접 Git 명령어를 칠 필요 없습니다.**

## 글 검토 + 발행 체크리스트

AI가 구글독스에 글을 생성하면:

1. **캘린더에서 확인**: 해당 날짜 이벤트 클릭 → 독스 링크
2. **내용 확인**:
   - 사실과 다른 내용이 없는지
   - 가격/서비스 정보가 맞는지
   - 브랜드 톤에 맞는지
3. **사진 추가**:
   - `[IMAGE: ...]` 표시된 곳에 적절한 사진 넣기
   - 시술 비포/애프터, 살롱 인테리어, 제품 사진 등
4. **발행**:
   - 아임웹 사이트에 올리기
   - 네이버 블로그에 올리기
5. **시트 업데이트**:
   > "봄철 두피케어 글 발행했어. 아임웹 URL은 https://... 이고 네이버는 https://... 야"
   
   → Claude Code가 시트에 URL 기록하고 상태를 `published`로 변경합니다.

뭔가 안 되면 Claude Code에게 그냥 말하세요:

> "동기화 안 돼. 에러 나와"

> "글 생성이 안 되는데 왜 그런지 봐줘"

> "시트에서 데이터를 못 읽는다는데?"

Claude Code가 에러를 확인하고 해결 방법을 알려줍니다.
