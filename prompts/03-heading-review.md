# 프롬프트 3: H2/H3 소제목 정리

위 콘텐츠 개요의 H2, H3 소제목을 검토하고, GEO 기준으로 불필요한 섹션을 찾아주세요.

단, 소제목은 임의로 삭제하지 말고, 삭제 여부를 사용자에게 먼저 질문하세요.

---

## [STEP 1] 소제목 진단

각 H2에 대해 아래 기준으로 평가하세요.

1. 검색자가 실제로 궁금해할 질문인가?
2. AI가 답변으로 사용할 수 있는 내용인가?
3. 문제 해결, 선택, 비교에 도움이 되는가?

각 소제목을 다음 중 하나로 분류하세요.

- **유지**
- **수정 권장**
- **삭제 후보**

이유는 한 줄로 간단히 설명하세요.

---

## [STEP 2] 삭제 여부 질문

삭제 후보로 판단된 소제목에 대해 다음 형식으로 사용자에게 물어보세요.

```
다음 섹션은 GEO 관점에서 활용도가 낮을 수 있습니다.
삭제할까요?

- H2: ○○○
  이유: AI가 답변으로 사용할 가능성이 낮음 / 서론 성격

→ 삭제 여부를 선택해주세요.
  (삭제 / 유지)
```

(사용자 답변을 기다립니다)

---

## [STEP 3] 최종 구조 재정리

사용자의 선택을 반영하여

- 삭제된 소제목 제거
- 수정 권장 소제목은 질문 형태로 개선 (필요한 경우)
- H2는 가능하면 검색자 질문 형태로 작성 (필요한 경우)

출력 형식:

```
최종 구조
H1:
H2:
  H3, H4는 꼭 필요한 경우에만 작성
```
