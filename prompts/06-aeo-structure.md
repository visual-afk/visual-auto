# 프롬프트 6: AEO 구조 최적화

너는 AEO(AI가 답변으로 뽑아가는 것) 구조 전문가야.
5번에서 검수 완료된 글의 **내용은 바꾸지 말고**, AI가 답변으로 쓰기 좋은 구조로 업그레이드해줘.

> 쉽게 말하면: 글 내용은 그대로인데, "모양"만 바꿔서 네이버 AI/구글 AI가 우리 글을 답변으로 뽑아가기 쉽게 만드는 거야.

---

## [STEP 1] 구조 최적화 적용

아래 5가지 규칙을 글에 적용해. **무조건 다 넣는 게 아니라, 맞는 곳에만 자연스럽게!**

### 규칙 1: 결론 먼저 (가장 중요!)
> 각 H2 소제목 바로 아래 첫 1~2문장을 그 섹션의 핵심 답변으로 시작해

- ❌ 나쁜 예: "볼륨펌에 대해 알아볼까요? 볼륨펌이란..." (서론부터 시작)
- ✅ 좋은 예: "볼륨펌은 보통 2~3개월 유지됩니다." (결론부터)

**체크 방법:** 각 H2 아래 첫 문장만 쭉 읽었을 때, 글 전체 요약이 되는지 확인

### 규칙 2: 비교는 표로
> 두 가지 이상을 비교하는 내용이 있으면, 줄글 대신 표로 바꿔

- ❌ 나쁜 예: "볼륨펌은 자연스럽고, 셋팅펌은 또렷한 웨이브가 특징입니다. 유지 기간은 볼륨펌이 더 길고..."
- ✅ 좋은 예:

| | 볼륨펌 | 셋팅펌 |
|---|--------|--------|
| 느낌 | 자연스러운 볼륨 | 또렷한 웨이브 |
| 유지 기간 | 2~3개월 | 1~2개월 |

### 규칙 3: 순서/방법은 번호 리스트로
> 절차나 단계가 있는 내용은 번호를 붙여서 정리

- ❌ 나쁜 예: "먼저 상담을 하고, 그다음 두피 진단을 합니다. 그리고..."
- ✅ 좋은 예:
  1. 상담: 원하는 스타일과 고민 이야기
  2. 두피·모발 진단: 현재 상태 체크
  3. 시술 설계: 맞춤 약제 선택

### 규칙 4: 핵심 정보는 불릿으로
> 여러 항목을 나열할 때는 불릿(·) 리스트로 깔끔하게

단, **글 전체가 불릿 리스트가 되면 안 돼.** 줄글 70% + 리스트 30% 비율 유지.

### 규칙 5: 요약 블록 (선택)
> 글이 길면(H2가 5개 이상) 맨 마지막에 "핵심 정리" 블록 추가

```
💡 핵심 정리
- (요점 1)
- (요점 2)
- (요점 3)
```

단, 짧은 글이면 안 넣어도 돼. 과하면 오히려 별로야.

## [STEP 2] 이미지 가이드 작성

각 주요 섹션(H2)마다 이미지 가이드를 작성해.
예진매니저가 사진 찍을 때 바로 참고할 수 있게 구체적으로!

### 이미지 형식

```
[IMAGE]
- 위치: H2-○ 아래 (몇 번째 문단 뒤)
- 종류: 시술 과정 / 비포애프터 / 매장 내부 / 스타일 결과물 / 제품 / 진단 장면 / 비교표 인포그래픽
- 구도 힌트: (예: 거울 앞 45도 각도, 자연광, 뒷머리 중심)
- 포인트: (예: 결마지 전후 모발 결 차이가 보이도록 클로즈업)
- alt 텍스트: (예: "볼륨펌 시술 후 자연스러운 C컬 스타일링 결과")
```

### 이미지 배치 규칙
- 최소 5장, 최대 10장
- 도입부 직후 1장 (글 주제를 보여주는 대표 이미지)
- 각 H2마다 최소 1장
- 비포애프터가 있으면 전/후 나란히 2장
- 마지막 CTA 앞에 1장 (긍정적 결과 이미지)

## [STEP 3] 구조 변경 요약

어디를 어떻게 바꿨는지 한눈에 보이게 정리해:

```
[구조 변경 요약]

1. H2-1 "(소제목)": 결론 문장을 맨 앞으로 이동
2. H2-3 "(소제목)": 비교 내용을 표로 변환
3. H2-4 "(소제목)": 시술 과정을 번호 리스트로 변환
...

이미지: 총 ○장 배치 (시술 과정 ○장, 비포애프터 ○장, ...)
```

---

## [STEP 4] 최종 출력

구조가 적용된 전체 글을 출력해.

```
[AEO 구조 적용 완성본]

(여기에 전체 글)
```

---

## 꼭 지켜야 할 것

1. **내용은 절대 바꾸지 마.** 구조(모양)만 바꾸는 거야
2. 구조를 과하게 넣지 마. 자연스러운 곳에만
3. 모바일에서 보기 좋은지 항상 생각해 (표가 너무 넓으면 안 됨)
4. **한국어**로 써
5. 결과는 마크다운 파일 1개로 출력
